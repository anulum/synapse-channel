# Extension support

Before reporting a problem, record the extension version, editor version, hub
version, connection state shown in the status bar, and the smallest sequence
that reproduces the issue. Never attach a token, credential store, private hub
database, or unredacted proprietary path.

- Report reproducible bugs through the
  [issue tracker](https://github.com/anulum/synapse-channel/issues).
- Ask workflow questions in
  [Discussions](https://github.com/anulum/synapse-channel/discussions).
- Follow the repository
  [security policy](https://github.com/anulum/synapse-channel/security/policy)
  for private vulnerability reporting.
- Read the extension [privacy notice](PRIVACY.md) and [licence](LICENSE).

The extension is an experimental preview. Support covers the documented local
VSIX and hub connection path. Registry operations remain owner-controlled;
confirm a registry's displayed publisher, version, and source before reporting
a registry-specific problem.
